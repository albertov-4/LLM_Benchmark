## SLURM Header Directives

All SLURM job scripts start with a standard header that defines resource allocation and job configuration:

```bash
#!/bin/bash
#SBATCH --account=IscrC_ArtLLMs       # Billing account (required)
#SBATCH --partition=boost_usr_prod    # Queue/partition to use
#SBATCH --time=00:30:00               # Maximum runtime (HH:MM:SS)
#SBATCH --gres=gpu:1                  # GPU allocation (optional)
#SBATCH --cpus-per-task=8             # CPU cores per task
#SBATCH --mem=32G                     # Memory allocation
#SBATCH --job-name=llm_test           # Job name for identification
#SBATCH --output=test_%j.out          # Standard output file (%j = job ID)
#SBATCH --error=test_%j.err           # Error output file (%j = job ID)
```

### Key Directives Explained:

| Directive | Purpose | Common Values |
|-----------|---------|---------------|
| `--account` | **Required billing account** | `IscrC_ArtLLMs` |
| `--partition` | **Queue selection** | `boost_usr_prod` (GPU), `boost_usr_prod` (CPU) |
| `--time` | **Maximum runtime limit** | `00:30:00` (30min), `02:00:00` (2h), `24:00:00` (24h) |
| `--gres` | **GPU resources** | `gpu:1` (1 GPU), `gpu:2` (2 GPUs) |
| `--cpus-per-task` | **CPU cores** | `4`, `8`, `16` |
| `--mem` | **Memory allocation** | `16G`, `32G`, `64G` |
| `--job-name` | **Job identifier** | Custom name for easy tracking |
| `--output` | **Standard output file** | `%j` expands to job ID |
| `--error` | **Error output file** | `%j` expands to job ID |

### Resource Guidelines:

- **Light tasks** (tests, validation): `--cpus-per-task=2 --mem=4G --time=00:05:00`
- **Model inference** (single run): `--gres=gpu:1 --cpus-per-task=8 --mem=32G --time=00:30:00`
- **Batch experiments**: `--gres=gpu:1 --cpus-per-task=16 --mem=64G --time=02:00:00`
- **Large-scale studies**: `--gres=gpu:1 --cpus-per-task=16 --mem=64G --time=24:00:00`

## SLURM Job Submission

### 1. Ready-to-Use Test Script

Use the provided script:
```bash
sbatch run_cluster_test.sh
```

### 2. Custom Test Jobs

Create your own job script:

```bash
cat > my_test.sh << 'EOF'
#!/bin/bash
#SBATCH --account=IscrC_ArtLLMs
#SBATCH --partition=boost_usr_prod
#SBATCH --time=00:30:00
#SBATCH --gres=gpu:1
#SBATCH --cpus-per-task=8
#SBATCH --mem=32G
#SBATCH --job-name=llm_test
#SBATCH --output=test_%j.out
#SBATCH --error=test_%j.err

# Setup environment
module load python/3.11.7
source venv/bin/activate
cd LLM-Needs-a-Plan

# Run tests
python3 src/tests/test_cluster.py
EOF

sbatch my_test.sh
```

## Job Management

### Submit and Monitor
```bash
# Submit job
sbatch my_test.sh

# Check status
squeue -u $USER

# Monitor in real-time
watch -n 5 'squeue -u $USER'

# Cancel job
scancel JOBID
```

### Check Results
```bash
# View output
cat test_JOBID.out # Outputs 
cat test_JOBID.err # Logs 

# List all outputs
ls -t *_*.out *_*.err

# Follow live output (for running jobs)
tail -f test_JOBID.out
```

| Test Type | Resources | Example |
|-----------|-----------|---------|
| **Quick Tests** | `--cpus-per-task=2 --mem=4G --time=00:05:00` | VAL integration, config tests |
| **Component Tests** | `--cpus-per-task=4 --mem=8G --time=00:10:00` | File manager, PDDL processing |
| **Model Tests** | `--gres=gpu:1 --cpus-per-task=8 --mem=32G --time=00:30:00` | Model loading, inference |
| **Full Pipeline** | `--gres=gpu:1 --cpus-per-task=16 --mem=64G --time=02:00:00` | Complete workflow tests |

## Common SLURM Commands

```bash
# Submit job
sbatch script.sh

# Check your jobs
squeue -u $USER

# Job details
scontrol show job JOBID

# Cancel job
scancel JOBID

# Live monitoring
watch -n 5 'squeue -u $USER'

# View outputs
cat output_JOBID.out
tail -f output_JOBID.out  # Follow live output

# View errors
cat output_JOBID.err
tail -f output_JOBID.err # Follow live errors 
```
