# Pre-Configuration Steps before Using Leonardo

## 1. CINECA Account Creation
- Make sure you have a valid account registered on the CINECA [UserDB](https://userdb.hpc.cineca.it).
- Verify you are part of the correct project allocation (e.g., `IscrC_ArtLLMs`).

## 2. Two-Factor Authentication
- Enable OTP (One Time Password) for secure access to CINECA systems.

## 3. Install `step-cli`
- On your local machine, install the `step` client to handle temporary SSH certificates.
- Documentation: [Smallstep CLI](https://smallstep.com/docs/cli).

## 4. Start the SSH Agent
Before requesting a certificate, start `ssh-agent` so that it can store and use the certificate:
```bash
eval "$(ssh-agent -s)"
```

## 5. Generate a Temporary Certificate
```bash
step ssh login 'your_email@domain.com' --provisioner cineca-hpc
```
- A browser window will open to authenticate via CINECA SSO.
- This generates a certificate valid for **12 hours**.
- You can verify the certificae was loaded 
```bash
ssh-add -L
```

## 6. Access Leonardo
Once the certificate is active, connect via SSH:
```bash
ssh <username>@login.leonardo.cineca.it
```
You will reach the login node (`loginXX.leonardo.local`).

## 7. Errors Accessing The Cluster 
The cluster login is active for 12 hours, then you have to repeart the overall process, but, before doing it, is recommended the following step in order to reboot the ssh key storage.

```bash
ssh-keygen -f "/home/<username>/.ssh/known_hosts" -R "login.leonardo.cineca.it"
```
