TASK FAMILY: block-grouping

You are solving a numeric PDDL planning problem.

The domain models blocks on a bounded 2D grid. Each block has numeric coordinates
x(block) and y(block). The goal usually requires blocks to reach target numeric
positions or to satisfy numeric coordinate relations.

Use only the action schemas defined in the domain:
- move_block_up
- move_block_down
- move_block_right
- move_block_left

Action interpretation:
- move_block_up increases y(block) by 1 and requires the new y value to be at
  most max_y.
- move_block_down decreases y(block) by 1 and requires the new y value to be at
  least min_y.
- move_block_right increases x(block) by 1 and requires the new x value to be at
  most max_x.
- move_block_left decreases x(block) by 1 and requires the new x value to be at
  least min_x.

Planning rules:
- Track x and y values for each block after every action.
- Respect the grid bounds min_x, max_x, min_y, and max_y before applying moves.
- Move only block objects that appear in the problem.
- Ensure all final numeric coordinate goals are satisfied.
- Do not output coordinate assignments or intermediate states.

Output rules:
- Output only executable actions.
- Do not output numeric assignments, numeric comparisons, effects, goals, or
  intermediate states.
- Do not output expressions such as (= ...), (>= ...), (increase ...), or
  (decrease ...).
- Each line must be one grounded action using only block names from the problem.
