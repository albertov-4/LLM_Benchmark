\subsection{Do the selected LLMs have reasoning capabilities?}
Being the datasets we gathered during our runs not complete due to: problems interfacing with NVIDIA API, Generation errors caused by the models internal procedures and some drawbacks encountered on Leonardo HPC due to wrong dependencies in the setup we can't give a definitve answer to this question.\\
However from the results we can see that some LLMs show partial reasoning-related behaviour, but not robust planning reasoning. Several models can produce syntactically valid PDDL actions and sometimes reach valid plans, especially with iterative repair; but low Success Rates, large retry gaps, and frequent validation failures show that they often fail to maintain a coherent world state over long action sequences. In the end the evaluated LLMs should be described as partial planners or repair-assisted generators rather than reliable autonomous planners.\\

\subsection{Why and How could our Benchmark answer the previous questions?}
The benchmark combines controlled PDDL tasks, formal validation, iterative repair and reasoning diagnostics; the selected domains cover different reasoning categories and difficulty levels, allowing model performance to be compared across both task type and complexity: each model output is stored in a traceable pipeline.\\
Also correctness is not judged from the apparent plausibility of the answer: A case is counted as solved only when the final submitted action sequence is formally valid and reaches the goal; iteration-based metrics then show whether the model solved the task immediately or only after repair feedback, while reasoning-text diagnostics show whether a valid plan was present in the reasoning trace but not correctly transferred to the final answer.

Therefore, the benchmark can measure both valid-plan discovery within a fixed iteration budget and the quality of the model's reasoning behaviour. It does not only report whether a model succeeds, but also helps explain how it succeeds or why it fails.
