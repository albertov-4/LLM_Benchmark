\section{General Structure}
The root folder contains various sub folders, specifically:
\begin{itemize}
    \item analysis
    \item archive
    \item Benchmark Framework
    \item results
\end{itemize}

\subsection{analysis}
This folder is used to gather all the datas useful for the production fo the report and the slides, contains the summary of the raw scores for the PDDL domains and the jupiter notebooks and the report.

\subsection{archive}
This folder contains all the material given us by the professor, specifically papers, a list of PDDL planning domains, two github repos given us as a reference and last a Quick PDDL guide written by us to help understand better the language.

\paragraph{docs} 
Contains a single file for the model preparation which i useful because the benchmark can run on a local PC or on an HPC system such as Leonardo. The important operational rule is that GPU benchmark jobs should not download large model weights. Prepare Hugging Face models first, then run benchmarks from local files.

\paragraph{evaluators}
Provides tools for to understand, validate and measure the capabilities of LLMs to solve planning problems

\paragraph{Leonardo script}
Contains specific scripts to launch and manage jobs on Leonardo HPC.

\paragraph{models}
Manages the communication with various LLMs, using "adapters" which creates a common interface for different backends like NVIDIA API.

\paragraph{outputs}
folder containing all the output from the LLMs, divided in 3 sub folders \textit{raw, parsed, scored} to better separate respectively the raw output from the model,PDDL extracted and strctured plans and validation and metrics results.

\paragraph{prompts}
contains text files each one used to explain to the LLM the rules of the domain and how to manage the problem in order to solve it.

\paragraph{protocols}
defines different strategies for test execution, for example direct plan (which is one shot) or iteratoive repair.

\paragraph{reporting}
contains scripts for creation of teh plots used in the reports.

\paragraph{runner}
contains scripts useful to launch a singlke test or the whole suite.

\paragraph{scripts}
contains scripts for the computation of the numerical score assigned to each domain.

\paragraph{slurm logs}
Used to gather information for debug when the benchmark runs on Leonardo HPC

\paragraph{tasks}
PDDL problems and related instances selected for the runs, the instances are divided in 3 categories \textit{easy, medium, hard}.

\paragraph{tests}
Used to run tests in preparation for the real run of the benchmark.

\paragraph{utils}
contains executable for the validator to be used during the run of benchmark framework

\paragraph{advanced planning evaluation.py}
it reads benchmark artifacts from the output folder and writes a model-centric JSON report in the folder results.

\paragraph{clear outputs}
is used to empty a folder during Leonardo tests

\paragraph{run benchmark} 
is the file that runs the whole sequence of tests\\

The framework and its structure is organized in this way in order to allow teh user to change fundamental parameters such as the numbers of iterations easily and effectively and then run the benchmark again with the modified parameters.
