\begin{frame}{Classification Purpose}
    \textcolor{blue}{\textbf{Purpose:}} In order to obtain reliable outputs to be evaluated after we for sure need a way to classify the PDDL problems to be fed to the LLMs chosen, since we had available 20 different problems a easy but effective way to classify them was needed to choose wisely the problems to be used.\\
    The main idea behind this Input Classification develops around two main concepts:
    \begin{itemize}
        \item  need of a numerical score representing the difficulty of the problem.
        \item classify problems based on the type of reasoning to be employed to solve the problem.
    \end{itemize}

\end{frame}

\begin{frame}{Implementation: Raw and Numerical Score}\footnotesize
    As stated before the idea of using a raw score to classify the difficulty of the problems came up because a relatively simple and repeatable way to evaluate problems was needed; in order to obtain the score the code in \textit{LLM benchmark/scripts/score domains complexity.py} has been designed.\\
    The main idea behind the computation of the score is to evaluate for each instance of a PDDL problem the presence or the absence of features that we believed could increment or decrement the difficuly of a problem, specifically:
    \begin{multicols}{2}
    \begin{itemize}
        \item N. of Objects
        \item N of possible Actions
        \item N. of Goals
        \item N. of Numeric goal conditions
        \item N. of Numeric functions
        \item N. of Numeric preconditions
        \item N. of Numeric effects
        \item Has metric
        \item Topology
    \end{itemize}
    \end{multicols}
    A raw score for each instance of a problem is therefore computed and then a normalization between instances of the same domain is employed; then for each domain a summary is presented.

\end{frame}


\begin{frame}{Implementation: Raw and Numerical Score II}\footnotesize
    The raw score for each instance is computed using the following formula:\\
    \resizebox{\textwidth}{!}{
        $\text{RawScore} = 0.45 \cdot \log(1 + \text{Nactions}) + 0.20 \cdot \log(1 + \text{Ngoals}) + 0.20 \cdot \log(1 + \text{num-score}) + 0.15 \cdot \text{topology}$
    }\\
    \vspace{0.2cm} % Spazio tra i due blocchi

    where the num-score is computed by:
        \resizebox{\textwidth}{!}{
        $\text{NumScore} =   1.0 \cdot (\text{N of num effects}) + 2.0 \cdot (\text{N of num preconditions}) + 3.0 \cdot (\text{N of num goal conditions})+ 1.0 \cdot (\text{has metric})$
    }\\
    the whole domain difficulty is the computed starting from the sum of all the raw scores obtained from the instances of the
    instances and then normalized 0/100.\\

    \vspace{0.2cm} % Spazio tra i due blocchi

            As it can be noticed other datas are added for completion sake, specifically the mean raw score of the instances and the median raw score.
        \end{column}

\end{frame}


\begin{frame}{Implementation: Reasoning Type} \footnotesize
    Another very important feature of PDDL problems is the fact that different problems may require different kind of reasoning to be properly managed; this said we defined 5 different reasoning categories to classify the problems in this sense:
    \begin{multicols}{2}
    \begin{itemize}
        \item Numerical optimization
        \item Temporal planning
        \item Logistic
        \item Logic
        \item Spatial
    \end{itemize}
    \end{multicols}


    Using these 5 categories we divided the problems available into these categories, it is worth noticiting that many problems might have mixed reasoning types, so we assigned the final reasoning label to a problem based on the type of reasoning required present most within the problem of interest.\\     
    In the end both using this classification and the difficulty score we end up obtaining a good classification able to take into account the main feature of the problems but still easy to deal with even for someone that is not familiar with this topic.
\end{frame}

    We selected 6 problems total: 3 of them are the hardest ones (one for each reasoning categories left) based on the difficulty score of the domains; the other 3 are the easiest ones, one from each reasoning categories left.\\

\begin{frame}{Hardware Platform}\footnotesize
    Now, since a good amount of computational resources is needed to run the LLMs we started looking for some resources and we ended up with tho different alternatives:
    \begin{multicols}{2}
    \begin{itemize}
        \item Leonardo HPC by CINECA
        \item NVIDIA API
    \end{itemize}
    \end{multicols}
    Both of them provided good amount of computational resources, more than sufficient run LLMs but the choice ended up being Leonardo HPC simply because with NVIDIA some problems came up since we needed a good internet connection and we didn't always have a clear feedback from the system.\\
    The CINECA provided us a trial project on Leonardo consisting of a Cluster composed by:
    \begin{itemize}
        \item 32 Cores
        \item 4 x A100 GPUs (64GB each)
        \item 500GB RAM
    \end{itemize}
    


\end{frame}

\begin{frame}{Single Evaluation Metrics sense and implementation}
    in this section will be decribed briefly the single evaluation metrics:
    \begin{itemize}
        \item Process Level Metrics ( reasoning quality )
        \begin{itemize}
        \item CoTaL ( chain of thought alignement)
        \item PAS ( precondition awarness score )
        \item ER ( executability ratio )
        \end{itemize}
        \item Efficiency Metrics ( reasoning efficiency )
        \begin{itemize}
        \item FASR ( first attemp succes rate )
        \item IWSR ( iteration weighted succes rate )
        \item RG = SR - FASR ( retry gap )
        \item P( valid | reached attemp k ) ( probability of getting a valid plan given that the attemp k has been reached )
        \end{itemize}
        
    \end{itemize}
\end{frame}
