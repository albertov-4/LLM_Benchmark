TASK FAMILY: fo-counters

You are solving a numeric PDDL planning problem.

The domain models counters. Each counter has a numeric value and a numeric rate.
Increment and decrement actions change the counter value by the current rate.
Rate-changing actions change the rate itself.

Use only the action schemas defined in the domain:
- increment
- decrement
- increase_rate
- decrement_rate

Action interpretation:
- increment increases value(counter) by rate_value(counter), and requires the new
  value to be at most max_int.
- decrement decreases value(counter) by rate_value(counter), and requires the new
  value to be at least 0.
- increase_rate increases rate_value(counter) by 1, and requires the new rate to
  be at most 10.
- decrement_rate decreases rate_value(counter) by 1, and requires the current
  rate to be at least 1.

Planning rules:
- Track value and rate_value for each counter after every action.
- Remember that changing a rate affects later increments and decrements.
- Respect bounds on value, max_int, and rate_value.
- total-cost may be increased by actions, but it is not itself an action.
- Ensure all final numeric goals are satisfied.

Output rules:
- Output only executable actions.
- Do not output numeric assignments, comparisons, effects, goals, or intermediate
  states.
- Do not output expressions such as (= ...), (>= ...), (increase ...), or
  (decrease ...).
- Each line must be one grounded action using only counter names from the problem.
